# 文本差异比较浏览器插件 - 设计文档

## 概述

一个基于 Manifest V3 的 Chrome 浏览器扩展，用于比较文本差异。支持多种文本类型（纯文本、JSON、XML/HTML、YAML）的格式化比较，提供双视图模式和多种输入方式。

## 项目结构

```
compare-diff-extension/
├── manifest.json              # Manifest V3 配置文件
├── background/
│   └── service-worker.js      # 右键菜单注册、消息中转
├── popup/
│   ├── popup.html             # 主界面
│   ├── popup.css              # 样式
│   └── popup.js               # 主逻辑：输入管理、视图切换
├── lib/
│   ├── diff.js                # jsdiff 库（文本比较引擎）
│   ├── formatters.js          # JSON/XML/YAML 格式化器
│   └── renderer.js            # 差异结果渲染（并排/内联视图）
├── content/
│   └── context-menu.js        # 右键菜单：选中文本快速比较
└── icons/
    ├── icon16.png
    ├── icon32.png
    ├── icon48.png
    └── icon128.png
```

**技术方案：** 原生 JS + jsdiff 库（无框架，与工作区现有插件保持一致）

**所需权限：** `activeTab`、`contextMenus`、`storage`

## 界面设计

### 输入阶段

- 顶部：比较类型选择栏（自动检测 + 手动切换：纯文本 / JSON / XML / YAML）
- 中间：左右两个文本输入框，支持粘贴文本和拖拽文件
- 底部：清空和比较差异按钮

### 结果阶段

- 工具栏：返回编辑、视图切换（并排视图 / 内联视图）、差异统计（+N -N ~N）
- 并排视图：左右两列对照显示，差异行用颜色高亮标记
- 内联视图：合并为单列显示，用颜色区分新增/删除/修改
- 颜色方案：绿色 = 新增，红色 = 删除，黄色 = 修改
- 底部：复制结果、复制差异摘要按钮

## 核心功能

### 1. 自动类型检测

根据内容特征按顺序检测：
- **JSON：** 以 `{` 或 `[` 开头，能被 `JSON.parse` 成功解析
- **XML/HTML：** 以 `<` 开头，包含标签结构
- **YAML：** 包含 `key: value` 缩进模式
- **纯文本：** 以上都不匹配时的回退类型
- 拖拽文件时额外检查文件扩展名（.json, .xml, .yaml/.yml, .html）

### 2. 格式化处理

| 类型 | 格式化行为 |
|------|-----------|
| JSON | 统一缩进（2空格），比较前对 key 排序，避免 key 顺序不同导致的误报 |
| XML | 美化缩进，标准化自闭合标签和属性顺序 |
| YAML | 统一缩进 |
| 纯文本 | 保持原样，可选统一空白符和换行符 |

### 3. 差异比较引擎

- 使用 `jsdiff` 库的 `diffLines` 进行逐行比较
- 修改行会进一步用 `diffChars` 标记具体修改的字符位置
- JSON 模式提供"忽略 key 顺序"选项
- 同时支持行级和字符级差异粒度

### 4. 右键菜单集成

- 用户在网页中选中文本 → 右键 → "添加到文本对比（左）" 或 "添加到文本对比（右）"
- 第一次选中设置为"原始文本"，第二次选中自动设置为"对比文本"并打开弹窗
- 未选中文本时菜单项置灰
- 通过 service worker 中的 `chrome.contextMenus` API 实现

### 5. 数据持久化

- 使用 `chrome.storage.local` 临时存储当前输入内容
- 重新打开弹窗时恢复上次的文本
- 不做历史记录存储，保持简洁

## 用户操作流程

1. **粘贴比较：** 打开弹窗 → 粘贴文本 → 点击比较 → 查看结果
2. **拖拽文件：** 打开弹窗 → 拖拽文件到输入框 → 点击比较 → 查看结果
3. **右键菜单：** 在网页中选中文本 → 右键"添加到对比（左）" → 再选中另一段文本 → 右键"添加到对比（右）" → 弹窗自动打开并填入文本 → 点击比较

## 项目范围

这是一个单一范围的项目，所有功能相互关联且规模适中，适合在一个实施周期内完成。
